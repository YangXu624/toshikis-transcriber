# AI Session Capture & Summarisation Tool

A modular, clean-architecture local-first Python application designed to capture audio sessions, transcribe them, structure the Q&A vs. presentation segments without altering content, and summarize the Q&A answers.

---

## 1. Pipeline Architecture & Directory Structure

The application is structured into **3 distinct, individually testable modules** located inside the [modules/](file:///Users/yangxu/code/toshikis_transcriber/modules) directory. Each module contains its own `input/` and `output/` directories. Downstream modules consume data from upstream modules via symbolic links.

```text
modules/
├── transcribers/
│   ├── input/                # Place raw audio here (e.g. agoda.mp3)
│   ├── output/               # Generates raw transcripts (e.g. agoda.txt)
│   └── run.py                # Runner script for Module 1
│
├── structurizers/
│   ├── input/                # Symlinked to modules/transcribers/output/
│   ├── output/               # Generates presentation & Q&A segments
│   └── run.py                # Runner script for Module 2
│
└── summarizers/
    ├── input/                # Symlinked to modules/structurizers/output/
    ├── output/               # Generates summaries (JSON & TXT)
    └── run.py                # Runner script for Module 3
```

### Automatic Directory Synchronization
To avoid copying files manually:
- [modules/structurizers/input](file:///Users/yangxu/code/toshikis_transcriber/modules/structurizers/input) is a symbolic link pointing to [modules/transcribers/output](file:///Users/yangxu/code/toshikis_transcriber/modules/transcribers/output).
- [modules/summarizers/input](file:///Users/yangxu/code/toshikis_transcriber/modules/summarizers/input) is a symbolic link pointing to [modules/structurizers/output](file:///Users/yangxu/code/toshikis_transcriber/modules/structurizers/output).

This guarantees that as soon as a file is generated by an upstream stage, it is immediately available to the downstream stage.

## 2. Core Design Philosophy

This project strictly adheres to **Clean Architecture** and the **SOLID** principles:
- **Zero Framework Leaks**: Core business logic and model definitions do not import or depend on libraries like Faster-Whisper, Gemini, Pydantic, or any external HTTP clients.
- **Interface Driven**: High-level orchestrators and CLI runners communicate with adapters strictly using Abstract Base Class interfaces.
- **Decorator Registry**: Adapters are dynamically registered at runtime. They can be added to the project without modifying any core orchestrator files.

For full architectural details, see [Architecture.md](file:///Users/yangxu/code/toshikis_transcriber/Architecture.md).

## 3. Getting Started

### 1. Requirements
- Python 3.12 or newer
- Virtual environment (recommended)

### 2. Installation
Set up your virtual environment and install the required dependencies:

```bash
# Set up a virtual environment
python -m venv .venv
source .venv/bin/activate

# Install requirements
pip install -r requirements.txt
```

### 3. Setup Configuration
Copy the environment template and set up your variables:

```bash
cp .env.example .env
```

Ensure your `GEMINI_API_KEY` is specified in `.env`.

> [!NOTE]
> The default model is configured as `GEMINI_MODEL_NAME=gemini-flash-lite-latest` to avoid quota limits (e.g. `gemini-2.5-flash` has extremely strict requests/day limits on free tier accounts).

## 4. Running the Modules

Each module's runner script can be run on **all files** in its input directory, or on **specific target files** to save time and API quota.

If a script is run on all files (no arguments), it will automatically skip any files that have already been processed in the output directory. If specific target files are provided as arguments, the runner will force a re-run for those files.

### Module 1: Transcriber
Runs Whisper locally using the fast `large-v3-turbo` model.

```bash
# Transcribe all new audio files in modules/transcribers/input
python modules/transcribers/run.py

# Transcribe only the agoda.mp3 file
python modules/transcribers/run.py agoda.mp3
```

- **Inputs**: `.mp3`, `.wav`, or `.m4a` files in [modules/transcribers/input/](file:///Users/yangxu/code/toshikis_transcriber/modules/transcribers/input)
- **Outputs**: Plain text transcripts (`.txt`) in [modules/transcribers/output/](file:///Users/yangxu/code/toshikis_transcriber/modules/transcribers/output)

### Module 2: Structurizer
Runs Gemini to separate the main presentation from the Q&A section without changing the wording. It adds speaker interaction labels (`[Question]`, `[Answer]`, `[Follow-up]`) to the Q&A section.

```bash
# Structure all new transcripts in modules/structurizers/input
python modules/structurizers/run.py

# Structure only the agoda.txt transcript
python modules/structurizers/run.py agoda.txt
```

- **Inputs**: `.txt` transcripts in [modules/structurizers/input/](file:///Users/yangxu/code/toshikis_transcriber/modules/structurizers/input)
- **Outputs**: `<name>_presentation.txt` and `<name>_q_and_a.txt` in [modules/structurizers/output/](file:///Users/yangxu/code/toshikis_transcriber/modules/structurizers/output)

### Module 3: Summarizer
Runs Gemini to generate structured summaries focusing **exclusively** on the Answers provided during the Q&A session.

```bash
# Summarize all new Q&A files in modules/summarizers/input
python modules/summarizers/run.py

# Summarize only the agoda_q_and_a.txt file
python modules/summarizers/run.py agoda_q_and_a.txt
```

- **Inputs**: `*_q_and_a.txt` files in [modules/summarizers/input/](file:///Users/yangxu/code/toshikis_transcriber/modules/summarizers/input)
- **Outputs**: Plain text summary (`_summary.txt`) and structured JSON (`_summary.json`) in [modules/summarizers/output/](file:///Users/yangxu/code/toshikis_transcriber/modules/summarizers/output)
