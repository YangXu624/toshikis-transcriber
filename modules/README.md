# Pipeline Modules

This directory contains the core pipeline stages of the transcription and summarization system. The system is designed with a decoupled, modular architecture to allow each stage to be configured, executed, and tested in isolation.

---

## 1. Modularity Concept

By separating the workflow into individual modules, we achieve:
- **Separation of Concerns**: Each stage does exactly one job (transcribe, structure, or summarize).
- **Quota Conservation**: Using APIs (like Gemini) on the free tier means rate limits can easily be hit. Breaking up the pipeline prevents run failures from requiring a full restart from scratch.
- **Easy Testing**: Each module's input and output files are clearly isolated, allowing for targeted developer verification.

---

## 2. Directory Structure of a Module

Each module under this directory follows a strict layout convention:

```text
modules/<module_name>/
├── input/          # Input files for the module
├── output/         # Location of output files generated by the module
├── run.py          # Independent CLI runner script
└── <adapter>.py    # Adapter code (e.g. wrapper around external SDKs)
```

---

## 3. Data Flow & Downstream Symlinks

Upstream output feeds directly into downstream input. To enable zero-copy, real-time synchronization, the input directories of downstream modules are **symbolic links** pointing to the output directories of their upstream predecessors:

```
[ Raw Audio Files ]
       │
       ▼
 ┌───────────┐
 │Transcriber│  ──► output/  (Raw .txt Transcripts)
 └───────────┘
                  ▲
                  │  (Symlinked: modules/structurizers/input -> ../transcribers/output)
                  ▼
 ┌───────────┐
 │Structurizer│ ──► output/  (separated *_presentation.txt & *_q_and_a.txt)
 └───────────┘
                  ▲
                  │  (Symlinked: modules/summarizers/input -> ../structurizers/output)
                  ▼
 ┌───────────┐
 │Summarizer │  ──► output/  (Structured *_summary.txt & *_summary.json)
 └───────────┘
```

---

## 4. The 3 Modules

1. **[transcribers/](transcribers/)**
   - **Inputs**: Raw audio files (.mp3, .wav, .m4a)
   - **Outputs**: Raw text transcripts (.txt)
   - **Engine**: Faster-Whisper (local offline model `large-v3-turbo`)
   
2. **[structurizers/](structurizers/)**
   - **Inputs**: Raw transcripts (.txt)
   - **Outputs**: Main presentation segment and speaker-labeled Q&A segment
   - **Engine**: Google Gemini (`gemini-flash-lite-latest`)
   
3. **[summarizers/](summarizers/)**
   - **Inputs**: Structured Q&A segments (`*_q_and_a.txt`)
   - **Outputs**: Narrated plain text summaries and structured JSON files
   - **Engine**: Google Gemini (`gemini-flash-lite-latest`)
